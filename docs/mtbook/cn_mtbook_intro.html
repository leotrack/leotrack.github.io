导论 ─ 网志与 Movable Type

这一章是说故事的章节。我们将概略地描述关于 Blog 这个玩意儿的来龙去脉；甚么是 Blog ？谁在 Blog ？我为什么要 Blog ？ Blog 能给我带来些甚么好处？为什么我要用 Movable Type 来 Blog ？ Movable Type 就祇能够拿来 Blog 吗？我们不但会在这一章里为大家解答这些问题，甚至还会为各位介绍 Blog 在华人世界发展的过程。希望所有的朋友也能藉由这一章的内容，而能够有一些初步的了解（或者是用来决定要不要继续浪费时间来读这份文件……）。

为什么你该注意到网志

首先让我们来看看最近发生的一些重要事件：

  • 全球最大的搜索引擎公司 Google 最近刚并购了 Pyra Labs ，这家公司的主要业务内容也就是提供免费网志空间 http://blogger.com
  • 哈佛大学与史丹佛大学相继提供全校师生免费的网志系统
  • CNN 自 2003 年三月十日起，开始报导关于网志的新闻

从这些事件中，我们不难嗅出这个叫做「网志」的玩意儿，似乎在台面下跃跃欲试，想要跳到桌子上吓你一跳！虽然亚洲一带的媒体有意或无意间都略过了关于「网志」的新闻，但是并不代表它在这里、在台湾就不会有影响力。

在网络的世界里，本来就没有甚么国界之分。能够被搜索引擎找到的，就是广大数据库王国的一员；谁能够被 Google 相中，谁就是当朝的红人。这个道理相当地简单，即使是还不识字的三岁小孩都能理解：要让别人听话行事的先决条件是，你得让他听到你在说些甚么。如果每天都有数以千万计的新页面在网络上冒出来、上亿的文字被使用，你又该如何让你说的话在这当中占有一席之地？你如何才能让你用心撰写的内容，顺利地呈现到你预期中的读者面前？

一种叫做「网志」的系统，一直潜藏于网络世界的幽暗面；如今这套系统逐渐在正式场合露脸。它让个人出版、个人新闻报导变得真实而容易，同时也让网页内容的提供者，能够轻易地维系著整个页面的枝干架构。它让文字的内容更能够贴近读者的目光，它唤起了无远弗届网络王国中的新脉动；无论你想要观察这个世界的心跳，或参与其中、或带头发难，你都不该忽略了这个新玩意儿。而这个称做「网志」的工具，也就是这份文件里所要加以阐述、剖析的。我们不但要告诉你它的惊人威力，更要揭露个中秘辛；甚至，我们还要告诉你，你该如何将这份力量化为己用！

网志发迹史

网志的原文是 Weblog ，或缩写成 Blog 。 Weblog 这个字眼最早是由 jorn barger 在 1997 年左右提出来的。在这之前，网络世界里所谓的 weblog 通常指的是 web log ，是一种硬生生地、充满技术性记载的、无关乎叙事的纪录。你可以把它想成是在网页服务器上的一个记录档，里面忠实地记载著这一台服务器上的众多封包往来：几点几分的时候，来自那个 IP 地址的计算机提出了甚么样的要求；几点几分的时候来自那个 IP 地址的计算机取得了些甚么样的数据；几点几分的时候出现了甚么错误讯息。诸如此类，这些内容既多且繁杂，更不具甚么可读性。若非是想要分析流量或察看可疑之处的系统管理人员，恐怕实在很难提起性子慢慢读（事实上，即便是这些系统管理人员，也都得仰赖其它的程序，才能「读」出这些记录背后的意义）。

那么 1997 年起出现的这个新字眼，又代表著甚么意思呢？所谓的 Weblog 指的是一种以 web 作为呈现媒介的（个人） log 。如果要从各位身边寻找比较接近的例子的话，或许相当于 BBS 上的个人板，或明日报个人新闻台吧。也有人把它称做网页型态的日记，不过没有人规定 weblog 每天一篇，所以或许称做网页型态的札记会更恰当。

到了 1999 年四、五月间， Peter Merholz 开始把 Weblog 念成 We Blog ；从此有了 Blog 这个说法。而且，这里的 Blog 不仅仅可以拿来当作名词、作为 Weblog 的缩写，更成为了一个动词。

当作名词的时候， Weblog ，或说是 Blog ，现在有了更严谨的定义；它指的是一些文章，而这些文章必须要满足这些条件：

  • 汇整
    这些文章必须经由特定的方法加以整理存放；这可以是单纯地按照时间来整理，也可以是任何的分类方式
  • 静态链结
    这些文章必须公开于网络之上，并且能够让其它读者藉由某个固定、不变的网址链结而直接读取得到；这暗指著这些文章的存取会是直接而公开的
  • 时间戳印
    这些文章必须具有时间戳印，记录著写成的时间
  • 日期标头
    这些文章必须要标出日期标头；这意味著它们将会是具有时序性的，或内容将可能与时空背景有一定的关联

于是 Blog 从很概略的「网页型式的个人札记」转变成具有多样特性的文章集结。而 Blog 这个字眼作为动词的时候，根据 Meg Hourihan 在 What We're Doing When We Blog 一文里的说法，乃是按照各自的规则来整理信息的方法。

根据这个说法，当某个人在 Blog 的时候，并不是（或不单纯祇是）生产被人们称之为「网志 (Blog) 」的一篇篇文章；实际上，她是在实践个人的信念，以她所认同且认可的规则，将她的感官所接受到的种种信息，整理后加以出版，成为其它读者所能够阅读的文本。

早在有网络之初，国外就有许许多多不同的人们，各自用了各种的技术在做这件事；至于华人世界里，最早开始接触 Blog 的也是那些侨居国外的学生。这些侨生由于身在国外，接受各种不同文化的刺激，于是她们也跟著仿效使用。由于她们使用的多半是国外的资源，所以几乎清一色地使用英文作为呈现文本的语言；虽然如此，这却是华人世界中，最早萌现的新芽。

随著近年来 Blog 逐渐受到重视，有一位在美国读书的菲律宾裔学生成立了「饭碗志」，作为亚裔网志大串联。她用米饭、用碗筷做为亚裔世界共通的语言：虽然我们可能用不同的语言、住在不同的国家，但是我们都是用碗筷吃米饭的亚裔，有著无以比拟的情感

单就两岸来说，台湾的信息介入显然丰富得多，但是除了 1996 年 PChome ePaper 电子报、 1997 年智邦电子报 、 2000 年明日报 MyPaper 个人新闻台这些加以品牌化经营的实际施用后，就呈现出一片停摆。反倒是近年来对岸开始起步了； 2002 年的八月与十月左右，相继出现了博客中国与中文Blog心得集这两个 Blog 集结网站 ─ 它们甚至开始回头探讨 Blog 这回事：是甚么？能怎么用？有些甚么可能性？稍晚，一样在 2002 年十月左右，这阵风才又吹回台湾。首先是由艺立协社群所带起了正体中文blog信息中心，半实验性地推广起 Blog ；接下来在 2003 年二月间，经由这些在台湾的推广活动之下，连带地出现了台湾应递媒与部落格实验与台湾部落格 - 天线部落 实验网络。而今历史正在改写，而阅读这份文件的你，也将成为这近代史上重要的一份支流。

从 Blog 到 Movable Type

前述提及，当 Blog 作为一种「方法」的时候，任何正常的人都会想要把心思著重在内容上，而不是其它旁枝末节的部分。然而出版网页所需的 HTML 语法、乃至于更复杂的文件架构（哪里是标题、哪一段有多重要等）本身，却是琐碎而无趣的。终日处理这些零星的技术部分，祇会让人们没办法把心意集中在内容的创作与书写上。于是一个又一个的套件或系统因应而生；它们或许看起来或用起来各有千秋，但是背后的目的都是一样的：让事情更简单、把计算机就能处理的事情交给计算机做。

例如像是 blosxom 就是一套有名的 Blog 系统。这个系统本身就祇有一个档案，相当的精简；它可以说是没有使用者接口，或者说，它拿别的程序的使用者接口来当作使用者接口。如果使用者懂得 Perl 这只程序语言，就能够在 blosxom 之上加入各式各样千奇百变的功能；如果你完全没有程序相关的基础，甚至更糟地连自己的机器也没有，那你还可以选择其它比较简单的解决方案。例如我们在前面提过的 blogger.com 就是一个很好的选择；她们甚至标榜说，祇要五分钟不到，你就可以让自己的 Blog 开张运作。

诸如此类的程序相当地多，像是 Radio Userland 、 GreyMatter 等都是。也有人列表比较过这些不同的 Blog 系统，你可以参考 http://www.udhien.net/blogtool/ 或 http://www.urldir.com/bt/ 上的数据（虽然它们都有些过时而没有被加以更新）。这些 Blog 系统甚至可以被当成是阳春的内容管理系统 (CMS) ，能够提供多使用者像是社群、社团、企业、社会运动人士使用；也能够降低技术门槛，让文件内容更容易索引与搜寻，让内容提供者专注于提供内容，而不需要为技术细节费心，相对地也下放了说话与书写的权力。另一方面，这样子的系统也有助于把内容与外貌分开来处理，所以每个页面都可以很快速地改头换面，呈现出新的风采。

而在这么多 Blog 系统之中，本书所特别要提出来的是 Movable Type 这一套最近才在第三届年度网志奖获选为年度最佳网志出版工具 (best web application for weblogs) 的系统。她们在自己的网站上如此陈述著：Movable Type 是一套去中心化的个人出版系统，这套网页接口的系统是设计来让定期更新内容之类的事情更加容易的。前述提及的「内容」可以是网志或线上新闻的文章项目、线上相簿里的照片、报纸站台的头条新闻、线上杂志的文章，或其它的东西。

Movable Type 的安装需求并不高，大致说来你祇需要备妥下列要求即可：

  • 能让你执行自己的 CGI 脚本的网页服务器空间/帐号
  • 前述的服务器上需要安装有 Perl ，且版本必须比 5.004_04 还要新
  • 前述的服务器同时要安装 DB_File 这个 Perl 模块，或 MySQL 及 DBD::mysql 模块
  • 你得要有用来上传必要档案的 FTP 软体
  • 你得要有能够使用 Javascript 和 cookies 的网页浏览器

祇要能够符合这些需求， Movable Type 几乎可以安装在各式各样的操作系统平台上。甚至，连 Windows 98 都能够安装 Movable Type 。

Movable Type 的优点

自 2001 年十月份面市以来， Movable Type 就一直在活跃地演化中；这套以 Perl 为核心语言的 Blog 系统最为强韧的地方或许正如其名所暗示的一般（ Movable Type 的意思是活字印刷术），也就是无限可能的弹性。许许多多的外挂模块，让 Movable Type 不论是在处理文字格式或者是内容处理应用上，都具备了广泛且快速的支持。

除此之外， Movable Type 还有众多优点，包括了：

  • 具弹性的数据储存
    你可以视实际的情况或需要，选择不同的数据库后端。像是 Berkeley DB 、 MySQL 、 PostgreSQL 或 SQLite 都在支持之列
  • 采用 XML-RPC API
    你可以搭配许许多多的用户端软体，像是 BlogBuddy 、 Bloggar 、 BlogApp 等。
  • 程序码均以函式库来处理
    这样你可以弹性地修改 Movable Type 的各部分功能，或在其它程序中叫用 Movable Type 的功能
  • 使用接口优雅精鍊
    操作简单直觉好上手；相信我，这绝对是最强大的部分
  • 具有引用功能
    能够让网志与网志间、文章与文章间发展出更多的紧密相连
  • 预设模版就有相当优异的 XHTML 与 CSS 水准
    光是采用系统附上的预设模版，就可以轻易产生出符合标准而且具有相当亲和力的页面了
  • 容易本土化
    Movable Type 里的各个讯息、描述文字等，都可以经由统一的本土化手续，而转为不同的语言
  • 具备公开搜寻引擎
    任何读者都可以藉由页面上的搜寻引擎，搜寻你所撰写过的文字内容，而毋须仰赖任何其它的搜寻引擎
  • 数据能够汇入与汇出
    因此你可以从别的系统中，把文章内容转移进 Movable Type 里；或者透过这个功能来备份你的网志内容
  • 能够同时按照多个模版产生多个档案
    这意味著你可以在一瞬之间，以相同的内容制作出好几份不同观感的页面
  • 能够建立多份网志
    所以祇需要安装一次，就可以拿来做各种不同的用途
  • 能够接受多位作者
    因此你可以拿来给一群人用
  • 文章能够分类
    每一篇文章都可以指派一个主要类别跟无限多个次要类别
  • 能够同时使用多种汇整方式
    再一次地，每当你新增一篇文章后，与之相关的各种汇整也都会一并更新
  • 内建回响系统
    所有的读者都能够用此功能来发表评论或回响
  • 还有许许多多....

除了上面提到的这些之外，更重要的是国内社群的支持。这一阵子里， Movable Type 在国内可以说是如火如荼般地被采用；同样地 Movable Type 在本土化的资源里也最丰富。无论是使用者之间的讨论分享、中文文件的撰写、甚至是更细节的程序码修改等，都有人在奉献著心力。同样地，当你遇到问题的时候，也最容易找到人来帮你解决各种疑难杂症 ─ 事实上你眼前的这份文件，恐怕也是开创华文世界的先例了吧！

基于以上种种原因，实在没有理由不用 Movable Type 不是吗？（另一方面，我相信这也就是为什么你会读这份文件的原因了）

Movable Type 的用途

那么人们都拿 Movable Type 来做些甚么事呢？

最简单也最基本的用途，莫过于拿来当作个人札记使用了。这就跟你用其它媒介所做的事情一模一样：你可以在任意时刻记录下附近发生的事、或你心里的想法；你不祇能够用文字，更可以使用图片或其它的媒体。这些事情是你在明日报个人新闻台或 BBS 的个人板里能做的，现在你仍旧可以在这里做。

除了让 Blog 环绕著你自己发展之外，也有人会拿特定的主题来 Blog 。例如说 Six Apart 自己就拿 Movable Type 来发布公司的公告，艺立协之前办 P3P 的时候，也拿 Movable Type 做了主题站台 http://p3p.elixus.org ； Schee 用 Movable Type 做了亚洲以华文探讨摩托文化的 Motomosa.com 。还有更多人也选择了 Movable Type ，像是专门讲电子化学习的新加坡站台 eLearningPost ，或者像是台北护理学院听语障碍科学研究所乾脆就用 Movable Type 来当作整个网站的管理后台。这些也都是适合 Movable Type 发挥的领域。

当然还有许许多多不同的用法；例如 Mark Pilgrim 就用 Movable Type 来写了一本讲网页亲和力的书 Dive Into Accessibility （中文翻译成「深入亲和力」）。值得一提的是这一本书也同时获选为第三届年度网志奖的年度最佳网志论文 (best article or essay about weblogs) 。另外像是 log of links 这个站台就是拿 Movable Type 来整理书签； The Morning News 则是拿 Movable Type 来当作新闻网站； What Do I Know 这个站台则是用 Movable Type 来整理图书；另外你也可以看到 Movable Type 的母亲 Mena G. Trott 在她自己站台上的 A Day Late ，实际上就是用 Movable Type 实做了月历……诸如此类。

大抵说来，祇要你要表达的页面会需要经常性的更新，或者单纯祇是在结构上有著某种一致性，就都很适合拿 Movable Type 来当作后台接口。不管是对内对外、半公开或全公开，都可以把琐碎的细节交给 Movable Type 来处理，祇需要把精力花在内容上就可以了。家庭、社团、行政机关甚至是社运团体，都能够从中获得不少益处。
