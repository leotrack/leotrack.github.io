第五章 ─ 管理类别

除了按照时间来把文章汇整归档之外，你也可以自己定义出不限数量的类别，然后把每一篇文章指派给若干类别；这其中的弹性相当之大，对于任何一篇文章来说，都可能处于下列状况之一：

  • 不属于任何类别
  • 祇属于某个类别，这个类别也称做这一篇文章的主要类别
  • 隶属于多个类别，其中必定有一个类别是这一篇文章的主要类别，而其它类别对于这一篇文章来说都祇是次要类别

而且你可以在任何时候重新修改文章所属的类别 ─ 你可以更换文章的主要类别、新增或取消次要类别的隶属关系、让本来不属于任何类别的文章加入某个类别、或让某一篇文章完全不属于任何类别。

这种分类的能力让你的网志得以更为清晰 ─ 或许你的兴趣十分广泛，时而从事文字创作、时而报导科技新闻，天文地理历史无一不包。现在你可以把文章放进不同的分类里，再加以汇整管理，让你的读者能够更方便第一次览尽所有相同领域的文章。

值得注意的是 Movable Type 此刻尚未支持巢状类别的架构；换而言之，虽然对于单一一篇文章来说，可以有主要类别与次要类别之分，但是这些类别实际上却是平等的 ─ 没有所谓的父类别与子类别的差别；某个类别可以是某篇文章的主要类别，却又是另一篇文章的次要类别。

当然你可以实做出「祇被当作次要类别」的类别以及「祇被当作主要类别」的类别，甚至你可以做出「两个名称一模一样的类别」，系统都不会弄混；但是别忘了，系统也不会晓得你在打的鬼主意，所以你得对自己的阴谋了如指掌才行。

本章我们祇会介绍一些跟类别有关的基本操作和注意事项，至于各种奇技淫巧则会留待第十章才来介绍。另外，由于你的网志一开始不会有任何预设的类别，所以 Movable Type 预设也不会启用分类汇整的选项 ─ 你得先有类别，才能够分类！关于分类汇整的设定方法及相关事项，也请参阅第八章的分类汇整小节。

新增类别

以下我们将为各位介绍新增类别的正常途径：

  1. 登入网志管理接口后，请如图一按下左方工具列上的「类别」按钮。

    图一：按下左方工具列上的「类别」按钮。
  2. 然后你会看到如图二的画面。在这个画面里会列出你既有的类别；因为我们还没有新增过任何类别，所以现在画面上应该甚么也没有。除此之外，你应该还会看到画面上有五个空白的字段，这五个空白的字段就是让你新增类别用的。请如图三把新类别的名称填入空白的字段里，一栏填一个，然后再按下底下的「储存」按钮。因为画面上就祇有五个空白字段，所以你在新增类别的时候，一口气最多也祇能新增五个；同时也请注意，由于在预设的情况下， Movable Type 会拿类别名称来当作分类汇整的汇整档名，所以你应该尽可能不要使用中、日、韩文、空白、以及问号（ ? ）、斜线（ / ）、反斜线（ \ ）、管道符号（ | ）、分号（ ; ）、星号（ * ）、大于符号（ > ）、小于符号（ < ）等特殊符号或 HTML 卷标来当作类别名称的一部份，否则很可能会出现预期外（或预期中）的错误。

    图二：编辑类别画面里，有五个空白字段，让你可以增加新的类别。

    图三：填入新类别的名称后，按下底下的「储存」按钮；在这个范例中我们打算要新增的三个类别分别叫做 first 、 second 以及 third 。
  3. 类别新增完成后，你会看到如图四的画面，按照类别名称排序列出了所有的类别，每一个类别底下还会注明目前已经有多少篇文章被指派到该类别，同时你也可以由此编辑类别属性。另一方面，无论你一次新增了多少类别，当你按下储存后就会发现，新增类别用的空白字段再度变成五个；就跟聚宝盆一样，你永远不用担心这些空白字段会有用完的一天。

    图四：我们刚刚输入的 first 、 second 以及 third 类别已经成功地被新增了；你可以看到目前还没有任何文章被指派到这些类别里。

    图五：无论你刚刚一次新增了多少类别，你会发现新增类别用的空白字段始终会恢复成五个。

临时新增类别

除了妥善规划后，循规蹈矩地用前一小节的方法新增类别以外，你也可以在新增或编辑文章的时候临时新增类别，这么一来你就不需要打断思绪先去新增类别再回来写作，也不需要先把文章写好才去新增类别以及调整文章所属类别。现在就让我们来看看该怎么做：

  1. 无论你是在新增文章或编辑文章的时候，主要类别下拉式选单的最后一项总是「新增类别...」，请如图六选择这一个。

    图六：新增文章或编辑文章的时候，如果打算临时新增类别，就从「主要类别」下拉式选单里选择「新增类别...」。
  2. 接下来会弹出一个如图七的窗口，内有一个让你新增类别的字段。请如图八输入新类别的名称，然后按下「加入」键。

    图七：这是新增类别用的弹出式窗口，有一个空白字段让你填写新类别的名称。

    图八：在空白字段里输入欲新增的类别的名称，然后按下「加入」按钮；这个范例里新增的类别叫 others 。
  3. 就这样，一个新的类别诞生了。同时你也可以注意到，你正在编辑或正在新增的那一篇文章，也会自动地把主要类别设定成你刚刚所新增的类别。

    图九：这篇新文章的主要类别被自动设定成我们刚刚临时新增的类别；在这个范例中，这个类别也就是 others 。

指派文章所属的类别

当你建立起各种类别之后，就能够把文章指派给各个类别了。你可以在大量编辑模式里一次指派多篇文章的主要类别，也能够在文章编辑画面中，指派单一文章的主要类别跟次要类别。现在就让我们逐一实做：

  1. 我们在第三章的时候介绍过大量编辑模式，在这个模式中，你可以一次修改多篇文章的各种字段，而这其中也包括了主要类别。你可以如图十使用每一篇文章的主要类别下拉式选单，修改该篇文章的主要类别，最后再按下储存就行了。

    图十：透过「大量编辑模式」，你就可以一次修改多篇文章的主要类别。
  2. 除此之外，你也可以在新增文章或编辑文章的时候，从「主要类别」下拉式选单里，调整该文章的主要类别。

    图十一：你在编辑文章的时候，就可以用「主要类别」下拉式选单，改变文章所属的主要类别。
  3. 除了设定主要类别之外，你也可以在新增或编辑文章的画面中，设定文章所属的次要类别；请如图十二按下主要类别下拉式选单旁边的「指定多重类别」链结。

    图十二：要指派次要类别的时候，请按下主要类别下拉式选单旁边的「指定多重类别」链结。
  4. 接下来会弹出如图十三的窗口，在这个窗口左边的字段里会列出目前存在于你的网志上（但没有被设定成这篇文章所属）的类别，右边的字段则是你已经指派给这篇文章的类别。请如图十四选取欲指定为该文章次要类别的类别名称；如果你想要一次选好几个类别的话，你可以按著 Control 键不放，再用鼠标逐一点选。然后再如图十五按下「 >> 」按钮，就可以把这些被选取的类别移动到右边的字段去，也就是使他们成为这篇文章所属的次要类别。另外，如果你打算取消某个次要类别的话，也可以如图十六从右方的字段里点选，一样也可以按著 Control 键不放再用鼠标一次点选多个类别，然后再如图十七按下「 << 」按钮，就可以把这些被选定的类别移动到左边的字段去，也就是取消这些类别归属。

    图十三：这是指派多重类别用的弹出式窗口，左方的字段会列出目前可以选择、但尚未成为该文章所属的类别名称清单，右方的字段则是已经被设定成该文章所属的类别名称清单。

    图十四：选择欲指派给该文章的类别，在这个范例中选取的是 first 和 third 。

    图十五：按下「 >> 」按钮就可以把被选取的类别移动到右边的字段去。

    图十六：选择欲取消的类别，在这个范例中选取的是 third 。

    图十七：按下「 << 」按钮，就可以取消所选的类别。
  5. 最后别忘了如图十八按下「储存」按钮，然后等到如图十九的画面出现后再按下「关闭」按钮来关闭这个弹出式窗口，这样就完成多重（次要）类别的指派了。

    图十八：按下「储存」按钮来储存目前选定好的类别指派关系。

    图十九：储存完成后按下「关闭」按钮来关闭这个弹出式窗口。

编辑类别属性

Movable Type 里的类别并不是祇有一个名称而已；对于每个类别来说，除了类别名称之外，你也可以任意地设定类别描述，而且每一个类别也都可以成为一个能够接受引用通告的引用对象，也能够设定整个类别的自动通告网址。其中后面两者我们将留待第七章连同引用功能一并介绍，在此我们祇先说明如何修改类别名称以及类别描述。

  1. 如图二十，在编辑类别画面中，你可以看到目前已经存在的类别，它们会按照名称依序列出，同时你也可以看到这些类别里各有几篇文章。当你打算修改类别名称的时候，就祇需要像图二十一那样直接修改类别名称字段的内容，然后再按下「储存」按钮即可。

    图二十：在编辑类别画面里，可以看到既有的类别及其所拥有的文章数量。在这个范例里我们可以看到除了 first 这一类里有一篇文章外， others 、 second 和 third 这三个类别都还没有包含任何文章。

    图二十一：我们打算把 first 类别改名为 last ，所以在这个范例中就直接把类别名称字段里的 first 给改成 last ，然后再按下「储存」按钮。
  2. 储存后你就会看到如图二十二的画面，不但类别名称更换了，而且原先属于该类别的文章也都跟著被改过来了 ─ 不需要重新指派文章分类，这真是太好了！

    图二十二： first 类别现在叫做 last 类别了，而且原来隶属于 first 类别的文章，也都隶属于 last 类别。
  3. 除了类别文章外，你还可以像图二十三那样点击类别名称后的「编辑类别属性」链结，来编辑其它的类别属性。

    图二十三：点击 last 类别后面的「编辑类别属性」链结，来编辑 last 类别的其它属性。
  4. 你会看到如图二十四的画面，你可以在此修改类别描述以及引用设定。我们现在祇有要修改类别描述，所以请按照图二十五那样任意地填写类别描述，然后按下「储存」按钮，就能把类别描述给储存起来了。填写类别描述的时候可以使用任意的字符，也就是说你可以在此使用中、日、韩文字，也可以使用各种特殊符号，甚至是 HTML 卷标都可以。换而言之，这就是做出特殊效果的关键所在了；我们不会在这一章讨论到这些华丽把戏，请参考第十章的说明。

    图二十四：这是 last 类别的属性编辑画面，可以修改的部分包括有类别描述以及引用设定。

    图二十五：在这个范例中，我们在 last 类别的类别描述中填写了「 The Last category 」，然后按下「储存」按钮。

    图二十六：类别描述被储存起来了。

删除类别

有朝一日你也会需要把不再有用的类别给删除。这种事情最有可能发生在你打算重新安排分类的时候 ─ 你会把某几类的文章并在一起，然后就会有一些再也用不到的类别了。

  1. 错误的作法是像图二十七那样把类别名称清除（改成空字符串）然后储存，这么一来类别并不会真的被清掉，而祇会像图二十八那样，冒出「没有名称的类别」。

    图二十七：这是错误的作法，把类别名称清空再存盘。

    图二十八：结果这个类别没有被清掉，而祇是变成没有名称的类别。
  2. 唯一能够删除类别的地方是从编辑类别画面中，如图二十九核选欲删除的类别后面的删除核选框，然后再按下「删除核选的项目」按钮，这个方法对前面那个「没有名称的类别」也有效。接著会弹出如图三十的窗口，要求你确认要把所选的类别删除；如果该类别里有一些文章的话，这些文章仍然会存在，祇是不再属于这个类别罢了。按下「删除」按钮后，你就会看到如图三十一的画面，说明这些类别都被删除了。

    图二十九：核选「删除」核选框后按下「删除核选的项目」按钮。

    图三十：按下「删除」按钮，确定要删除这些类别。

    图三十一：刚刚被选取的类别都被删除了。
