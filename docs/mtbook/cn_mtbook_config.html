第二章 ─ 设定 Movable Type

当你按照第一章完成 Movable Type 安装后，还需要先加以调整各项设定，然后才能够开始使用。

以下我们将逐一加以说明，请务必按照本章指引来调整 Movable Type ，否则可能会发生意料之外的错误。

设定 mt.cfg

大部分的设定值都写在一个叫做 mt.cfg 的档案里；无论你是在甚么操作系统上安装了 Movable Type ，都会需要修改这个档案内的设定。以下的步骤原则上是写给 Windows 的使用者看的， FreeBSD 、 Linux 或其它 UN*X 操作系统的使用者，请用任何文字编辑器编辑 Movable Type 安装目录底下的 mt.cfg （例如 /usr/local/www/cgi-bin/mt.cfg ），然后一并按照步骤三之后的说明来加以修改。

  1. 执行开始 » 程序集 » 附属应用程序 » 记事本 来开启 记事本 。
  2. 如图一，开启 C:\Program Files\OurInternet\Movable Type\mt\mt.cfg 这个档案。
    请注意，你可能得在档案类型里选择所有档案，才有办法看到这个档案。

    图一：在 记事本 中开启 C:\Program Files\OurInternet\Movable Type\mt\mt.cfg 。
  3. 如图二，在这个档案里找到写著以下内容的那一列：
    CGIPath http://localhost/

    图二：在 mt.cfg 里找到写著 CGIPath http://localhost/ 的那一列。
  4. 在步骤三我们所找到的那一列，是用来设定 Movable Type 的 CGI 路径的。这一个设定项目如果没有设好的话，将会导致你的网页读者没办法顺利使用回响及引用功能。如果你按照了第零章／免费的网域名称小节的说明申请了免费网址，或按照第零章／浮动 IP 的解决方案小节的说明申请了免费的浮动网域名称的话，你就应该把这一列中的 localhost 改成你所设定的网域名称（如果是免费的网域名称的话，就是在步骤十一；如果是浮动网域名称的话，则是在步骤十一），最后面别忘记加上一个 / 符号，如图三。如果你是 UN*X 平台（如 FreeBSD 或 Linux ）的使用者，或者你重新修改过网页服务器，把 CGI 路径指到别的目录的话，你也该在这里做出相对应的修改，例如把这一列改成 CGIPath http://Jedimt.dyndns.org/cgi-bin/ 。

    图三：修改 CGIPath 的设定值。在这个范例中，我把这一列改成 CGIPath http://Jedimt.dyndns.org/ 。
  5. 接下来这两个步骤祇有 Windows 的使用者（或者需要用 SMTP 来寄信的 UN*X 使用者）才会用到。一般 UN*X 的使用者请直接跳到步骤七；另外也请确定你已经按照第一章的指示安装了 Mail::Sendmail 模块，接下来这几个步骤所做的修改才会有效。请继续在 mt.cfg 里寻找以下两列的内容，如图四：
    # MailTransfer smtp
# SMTPServer smtp.your-site.com

    图四：在 mt.cfg 里寻找跟 SMTP 有关的设定。
  6. 对于 Windows 的使用者来说，要寄信祇得用 SMTP 协议，因此我们得修改步骤五找到的那两列。请先把「 # MailTransfer smtp 」这一列前面的「 # 」删除，如图五；然后「 # SMTPServer smtp.your-site.com 」这一列中，除了也要把「 # 」删除之外，还得把「 smtp.your-site.com 」修改成你能够使用的送信主机。举例来说，如果你用的是 HiNet 的 ADSL 的话，你就可以把 smtp.your-site.com 改成 ms1.hinet.net ，如图六。

    图五：启用 Movable Type 的 SMTP 功能。

    图六：设定 SMTP 服务器。在这个范例中设定的是 ms1.hinet.net 。
  7. 让我们继续在 mt.cfg 往后寻找。你应该要看到有一列写著 NoHTMLEntities 1 ，如果这一列前面有 # 的话，请加以删除，务必确定这一列祇有以下的内容，如图七，否则你将会在文章编辑字段中看到一堆乱码：
    NoHTMLEntities 1

    图七：不要把内容编码成 HTML 本体。
  8. 如果你的服务器位于防火墙之后，而必须使用另外的代理服务器 (Proxy Server) 才有办法联机出去的话，你就得如图八自己加上这一列指令：
    PingProxy http://代理服务器地址:代理服务器连接埠
    这一列指令可以加在 mt.cfg 里的任何位置，如果你在储存文章时发生了「发送引用通告出错」的错误讯息，那么你就该设定这一项指令了。

    图八：设定引用功能会用到的代理服务器，在这个范例中我们设定的是「 http://proxy.hinet.net:80 」。
  9. 继续在 mt.cfg 往后寻找，看看有没有一列写著 PublishCharset UTF-8 ；这是用来处理万国码 (UNICODE) 中文字的。如果你找不到这一列的话，请按照图九，自己加上这一列：
    PublishCharset UTF-8

    图九：设定出版字元集设定为 UTF-8 。
  10. 现在我们要存盘了。请如图十从档案选单里选择储存盘案，最后再如图十一按下关闭图标即可。恭喜你完成设定了。

    图十：储存盘案。

    图十一：按下 记事本 的关闭图标。

设定数据库

Movable Type 如果不加以调整设定的话，会用 Berkeley DB 来当作数据库后端；实际上对于一般的使用者来说，这样也就够了。但是如果你决定要用 MySQL 或 PostgreSQL 来当数据库后端的话，就必须按照本节的内容加以设定才行。

如果你是 Windows 的使用者，而且用了我们在第一章提到的安装档的话，就可以直接跳过这一个小节，因为这个安装档里，使用的是 Berkeley DB 。除非你自己另外装了 MySQL ，才有必要继续读这一节。

设定数据库所会动到的档案祇有两个，分别是 mt.cfg 和 mt-db-pass.cgi ，请按照以下的步骤逐一修改其内容：

  1. 用文字编辑器开启 mt.cfg 。
  2. 在 mt.cfg 里加入这一列：「 ObjectDriver 驱动程序_名称」。这里的「驱动程序_名称」可能会是「DBI::mysql」（如果你用的是 MySQL ）、「DBI::postgres」（如果你用的是 PostgreSQL ）或「DBI::sqlite」（如果你用的是 SQLite ）
  3. 在 mt.cfg 里加入这一列：「 Database 数据库_名称」。这里的「数据库_名称」指的是你要用来存放数据的 SQL 数据库名称。
  4. 在 mt.cfg 里加入这一列：「 DBUser 数据库_使用者」。这里的「数据库_使用者」指的是你要用来连接到数据库的使用者名称。
  5. 如果你的 SQL 服务器位于不同的主机上的话，也请在 mt.cfg 里加入这一列：「 DBHost 数据库_主机」。这里的「数据库_主机」指的是你的 SQL 主机名称。
  6. 如果你的 SQL 服务器位于本地端，但是 socket 却不是在标准的位置上时，则请在 mt.cfg 里加入这一列：「 DBSocket /路径/指到/mysql.sock 」。这里的「 /路径/指到/mysql.sock 」指的是你的 SQL 的 socket 路径。
  7. 储存并关闭 mt.cfg 。
  8. 接下来的步骤祇有用 MySQL 或 PostgreSQL 的使用者才需要设定；如果你用的是 SQLite 的话，可以直接跳到下一节去。现在请用文字编辑器开启 mt-db-pass.cgi 。
  9. 在这个档案里祇需要写一列，这一列的内容就是你的 SQL 数据库口令。
  10. 储存并关闭 mt-db-pass.cgi 。这样就完成数据库设定了。

第一次激活系统

如果你是 Windows 的使用者，而且用了我们在第一章提到的安装档的话，一样可以直接跳过这个小节，因为安装程序已经帮你做完这件事了。

对于其它平台的使用者、或者选择手动安装的使用者来说，你的 Movable Type 还不能正式开始运作；我们还得先把数据库的架构准备好，才能够开始使用 Movable Type ，否则连预设的使用者帐号口令都没有。请按照以下的步骤：

  1. 用浏览器连到 Movable Type 的 mt-load.cgi 。举例来说，如果你的网页服务器位于 http://Jedimt.dyndns.org/ ，而 CGI 脚本放在 cgi-bin/ 的话，那你就应该要连到 http://Jedimt.dyndns.org/cgi-bin/mt-load.cgi 去。
  2. 你应该会看到如图十二的画面，这个时候就表示数据库初始化完成了。请按下浏览器里的「进入设定画面」链结，并继续参考下一节的内容组态你的网志。
    注意：一旦你完成数据库的初始化后，你最好把 mt-load.cgi 这个档案给删除掉，以免发生预期外的安全性漏洞。

    图十二：数据库初始化完成了，请按下「进入设定画面」的链结。

网志组态

虽然我们已经完成了 mt.cfg 的设定，也把数据库的架构建起来了，但是此时 Movable Type 内的各项设定值都还是预设值，你应该按照本节的内容加以调整，才能够符合你自己的实际设置状况：

  1. 如果你还没有登入 Movable Type ，请用浏览器连到 Movable Type 的 mt.cgi 。举例来说，如果你的网页服务器位于 http://Jedimt.dyndns.org/ ，而 CGI 脚本放在 cgi-bin/ 的话，那你就应该要连到 http://Jedimt.dyndns.org/cgi-bin/mt.cgi 去；把 Movable Type 安装到 Windows 的使用者则应该是连到 http://Jedimt.dyndns.org/mt.cgi 之类的网址去，因为 Windows 的安装档会把 CGI 脚本放在网页服务器的 / 里。
    如果你早就登入 Movable Type 了的话，请直接跳到「核心设定」的部分。
  2. 第一次登入系统的时候，请使用预设的帐号来登入。如图十三，预设的使用者帐号 (Username) 是 Melody ，而预设的口令 (Password) 则是 Nelson ；依图填妥后，请按下「登入 (LOG IN)」按钮。
    注意：如果你有在用自然输入法的话，请务必不要把自然输入法设成预设输入法，否则会发生登入错误的状况。

    图十三：用 Melody/Nelson 的使用者帐号/口令来登入 Movable Type 。
  3. 登入 Movable Type 之后，你应该会看到如图十四的系统主画面；请先按下画面上的「管理网志」链结，再依图十五按下左方工具列上的「网志组态」按钮，然后参照「核心设定」的部分加以设定。

    图十四：这是 Movable Type 的系统主画面，请按下「管理网志」的链结。

    图十五：请按下左方工具列上的「网志组态」按钮。

核心设定

  1. 在「网志名称」这一栏里，设定的是你的网志名称。你可以把它想成就是一个站台的名称，例如「飞雪连天电子报」之类的东西。请依图十六把预设的「我的网志」改掉。

    图十六：设定网志名称。在这个范例中所填入的是「光剑格斗学院」。
  2. 「本机站台路径」指的是说你要把 Movable Type 产生出来的网页档案放在硬碟上的那个地方；除非你知道你自己正在做些甚么，否则你最好不要乱动这一栏的设定值。
  3. 「站台网址」指的是你的索引页面网址（不含档名）；对于把 Movable Type 安装在 Windows 上的使用者来说，祇需要把 localhost 这个字符串替换成你申请到的网域名称就可以了；这也就是你在第零章「免费的网域名称」小节步骤十一设定的网域名称，或「浮动 IP 的解决方案」小节步骤十一设定的动态网域名称。
    请依图十七把 localhost 改成你的网域名称，并务必保留前面的 http:// 和最后的 / ，否则别人将难以浏览你的网志。

    图十七：填写站台网址。在这个范例中填写的是 http://Jedimt.dyndns.org/ 。
  4. 「本机汇整路径」指的是说你要把 Movable Type 产生出来的网页汇整档案放在硬碟上的那个地方；除非你知道你自己正在做些甚么，否则你最好不要乱动这一栏的设定值。
    理论上这一栏可以填得跟「本机站台路径」一样，不过这么一来，你的「汇整网址」就也得要跟「站台网址」设成一样才行。
  5. 「汇整网址」指的是你的汇整页面网址（不含档名），这个网址应该会跟你的「站台网址」有关才对；对于把 Movable Type 安装在 Windows 上的使用者来说，祇需要把 localhost 这个字符串替换成你申请到的网域名称就可以了；这也就是你在第零章「免费的网域名称」小节步骤十一设定的网域名称，或「浮动 IP 的解决方案」小节步骤十一设定的动态网域名称。
    请依图十八把 localhost 改成你的网域名称，并务必保留前面的 http:// 和最后的 / ，否则别人将难以浏览你的网志。

    图十八：填写汇整网址。在这个范例中填写的是 http://Jedimt.dyndns.org/archives/ 。
  6. 把画面往下卷动，「时区」这一栏应该是个下拉式选单；你应该从这个下拉式选单里选择你本人所在地的时区（而不是服务器所在位置的时区）。对于住在台湾的使用者来说，你应该如图十九选「UTC+8 (中国沿海, 台湾)」。

    图十九：从时区中选择 UTC+8 (中国沿海, 台湾) 。
  7. 上述的设定都修改过之后，请如图二十按下「储存」按钮。

    图二十：按下「储存」按钮。
  8. 设定储存起来后，你应该会看到如图二十一的画面。现在我们要继续调整别的设定，请按下「偏好」链结，然后参照下一小节的指引。

    图二十一：设定储存后，按下「偏好」链结。

偏好

在这个页面里的设定比较多，大致又可以分成四类：一般设定、汇整、出版/远程接口/引用、以及回响组态。让我们一类一类地加以设定：

一般设定

  1. 「网志描述」这一栏是用来填写你的网志的描述用的。一般说来，这里可以用来填写像是标语或口号之类的东西，例如「和平、奋斗、救中国」。这一栏可以留空不填，也可以如图二十二这样填入任意的文字内容。

    图二十二：填写网志描述，在这个范例中填写的是「如光似影／人剑合一」。
  2. 「文章显示天数范围」指的是你要在首页（也就是主索引页面）放多少天内的新文章。这个值祇是一个预设值而已，你随时都可以修改模版，来做出不同的效果，像是改成祇放最后几篇新文章之类的。你应该视文章实际的新增频率来调整这里的数值：如果你的文章增加得很快的话，就应该设小一点，反之就应该设大一点。如果你完全不知道该设成多少的话，按照图二十三这样保留预设值 7 天或许也是个好主意。

    图二十三：设定文章显示天数范围，在这个范例中我们保留 7 天的预设值。
  3. 「显示日期用的语言」会影响到你的页面中任何日期的显示方法。 Movable Type 截至目前为止还没有做出中文日期的显示格式，所以请如图二十四这样在下拉式选单中选择「英语」。

    图二十四：设定显示日期用的语言，请依图标选择英语。
  4. 「文章显示顺序」会影响到任何有两篇（或更多）文章同时出现的页面中，文章排列的顺序。祇要你的文章更新频率没有太低的话，你就应该如图二十五照预设值选「递减」，这么一来新的文章才会出现在页面的开端，（假想中的）读者一旦能够轻易地发现你的新文章，就更会常常晃过来看你的东西。

    图二十五：设定文章显示顺序，在这个范例中我们保留了递减的预设值。
  5. 「Default Text Formatting for new entries （新文章的预设文字格式）」指的是发表文章时的预设文字格式。除非你有特殊需求，不然你应该如图二十六这般，用预设值「转换换列」来自动处理文章中的段落（自动加上 <p> 与 </p> ）及换列（自动加上 <br /> ）。

    图二十六：设定新文章的预设文字格式，在这个范例中我们保留了转换换列的预设值。
  6. 文章的出版状态如果是「草稿」的话，就不会显示在网页上；祇有出版状态为「定稿」的文章才会显示出来。「预设的文章出版状态」就是在设定文章出版状态的预设值的。当然你在发表新文章或编辑旧文章的时候，仍然可以随时改变出版状态；但是如果你没有特殊需要的话，在这里就按照图二十七设定成「定稿」显然会方便许多。

    图二十七：设定预设的文章出版状态，在这个范例中我们设定成「定稿」。
  7. 新增文章的时候，可以在文摘字段里填写这一篇文章的文摘；如果你没有填写那一栏的话， Movable Type 就会拿「主要文章本体」这一栏的前几个字来当作文摘。这里的「文摘字数」就是在设定这个的。如果你不知道要设成多少，就如图二十八保留预设值 20 个字吧。

    图二十八：设定文摘字数，在这个范例中我们保留了 20 个字的预设值。
  8. 「消毒规格」会决定回响里可以用的 HTML 卷标。如果你甚么都不懂，就如图二十九这样，选「按照预设值」吧。

    图二十九：设定消毒规格，在这个范例中我们选了「按照预设值」。
  9. 图三十里的「欢迎讯息」大概祇有在同一份网志有许多不同作者一起维护的时候，才会有用，所以我们在此就先不写了。至于「创意公用授权条款」的部分，请见附录二的介绍；如果你对于甚么是创意公用授权条款完全没概念的话，现在最好啥也别碰。

    图三十：欢迎讯息与创意公用授权条款。在这个范例中我们甚么事也不做。

汇整

这一个区块中的设定选项都与汇整 (archiving) 有关，你可以一并参照第八章的介绍。

  1. Movable Type 除了能让你很容易新增文章之外，也会按照你指定的方法自动地帮你把文章蒐集起来汇整归档，这一个「偏好汇整类别」就是让你设定你最惯用的汇整方法的。在多数的情况下，你应该会想要跟图三十一一样，选择「单篇汇整」；这么一来，从你的首页（也就是主索引页面）点选任何一篇文章的时候，就会连到祇有该篇文章的汇整页面去。

    图三十一：设定偏好汇整类别，在这个范例中我们用了单篇汇整的预设值。
  2. 「汇整档案扩展名」是让你设定产生出来的网页档案扩展名的。如果你懂得 PHP 或其它各种语言的话，就能够善用这个设定；除此之外，如图三十二用预设值 html 显然会是个好主意。

    图三十二：设定汇整档案扩展名，在这个范例中我们保留了 html 这个预设值。

出版/远程接口/引用

  1. 「当我更新我的网志时通知下列网站」选项跟「通告 (ping) 」功能有关：有一些服务器，像是 blo.gs 和 weblogs.com 都会列出近期更新过的网志（或网站）。这些服务器当然不可能整天花费大量的力气去察看全世界的的网站页面有没有更新，他们之所以能够知道页面有所更新，是因为他们能接受及处理更新通告。
    Movable Type 就能够送出这种更新通告，所以为了打响你的知名度，请按照图三十三核选 blo.gs 和 weblogs.com 这两个核选框吧。如果你还知道其它服务器也能处理这种更新通告的话，也可以再填进下面那一个字段里，并以换列作为间隔。

    图三十三：核选 blo.gs 和 weblogs.com ，让你的网志更广为人知。
  2. 「近期更新金钥」祇对有捐钱给 Six Apart （这是 Movable Type 的公司）或购买了 Movable Type 商用许可的使用者有用；在这里填入了近期更新金钥后，你的 Movable Type 才会发送更新通告给 Movable Type 的官方网站。
    对于一般的使用者来说，留著空白不填就可以了。
  3. 「Google API 金钥」祇对有用到 Google API 的人才有用。如果你会用到 Google API 的话，请先去 Google 申请一份 Google API 金钥，然后填到这里来。
    对于一般的使用者来说，留著空白不填就可以了。
  4. 「预设允许通告」选项跟引用功能有关，你可以参考第七章的介绍。大致上来说，设定了这个选项之后，能够让读者们更容易接触到与你的文章相关的其它文章，也比较能够让你的网志与其它人的网志间有更好、更频繁的互动。所以强烈建议你按照图三十四核选这个核选框。

    图三十四：核选预设允许通告的核选框。
  5. 「用 Email 通知新的引用通告」选项一旦被核选之后，祇要有人引用了你的文章， Movable Type 就会寄一封信通知你。不过在平常的状况下，这是没有必要的，因为祇要有合适的模版，你连到自己的网志上的时候就自然会看到了嘛。有谁会在没有特殊需求的情况下，还希望一直收到信件打搅呢？
    这个选项跟引用功能有关，你可以参考第七章的介绍。
  6. 「启用引用自动发掘功能」选项跟引用功能有关，你可以参考第七章的介绍。这个选项强烈建议核选，因为一旦核选之后，祇要你的文章中有其它网站的链结， Movable Type 就会自动送出引用通告；简单来说，就是自动帮你在别人的文章里打广告。
    这种这么方便的功能，不用实在太对不起自己了。所以请参照图三十五核选这个核选框。

    图三十五：核选「启用引用自动发掘功能」的核选框。

回响组态

这一个区块中的设定选项都跟回响 (comments) 功能有关，你可以一并参照第六章的介绍。

  1. 一旦你勾选了「允许匿名回响」的核选框后，读者发表回响时就可以完全不需要填写姓名（昵称）、 Email 地址以及网址。虽然说，就算你不要核选这个核选框，也没办法阻止读者发表回响时留下假数据，但是这么做还是有点好处的：因为读者可能祇是「忘记填写基本数据」而已，而如图三十六保留这个核选框不勾选的话，就会在这种情况下提醒读者说，「啊，你忘记填写基本数据了呦！」

    图三十六：不要核选「允许匿名回响」的核选框。
  2. 「回响显示的顺序」会决定同一篇文章里，回响呈现的先后次序。在文章显示顺序的设定中，我们提到了文章排列时，越新的应该放在越前面才方便读者阅读，不过对于回响来说，情况却不同。
    因为文章可以祇按照时间来透露重要性（越近期被写下的，越能代表作者最近的心境），但是跟著某一篇文章的回响，彼此间却会出现脉络性，所以越新的回响应该出现在越后面，其它读者才能够循著脉络读下去。
    基于这个原因，我们按照图三十七保留预设的递增。

    图三十七：设定回响显示的顺序，在这个范例中我们保留了预设的递增。
  3. 「有新的回响时寄给我」核选框就跟「用 Email 通知新的引用通告」核选框一样，在正常的情况下实在不需要加以核选，因为你祇要浏览自己的网志，顶多略微修改一下模版就够了。何必跟自己的信箱过不去呢？
  4. 「回响文字格式」指的是读者发表回响时的文字格式。除非你有特殊需求，不然你应该如图三十八这般，用预设值「转换换列」来自动处理回响中的段落（自动加上 <p> 与 </p> ）及换列（自动加上 <br /> ）。

    图三十八：设定回响文字格式，在这个范例中我们保留了预设的转换换列。
  5. 如果你核选了「允许回响使用 HTML 语法」的核选框，那么要发表回响的读者就可以在回响中使用各种 HTML 卷标。这么做的好处是，读者发表回响时也可以有很大的自由度，但是相对来说，她们甚至有机会在你的页面里加入奇怪的背景音乐，或者其它可能会危及其它读者的动作（当然你还是可以透过消毒规格来加以限制她们可用的 HTML 卷标）。因此在图三十九的范例中，我们并不核选这个核选框。

    图三十九：不要允许回响使用 HTML 语法。
  6. 祇要你没有允许回响使用 HTML 语法，你就应该要如图四十核选「自动连结网址」核选框。这么一来， Movable Type 就会把任何在回响里出现的网址，通通转换成可以直接按下的超级链结，于是读者就不需要自己再复制一次网址了。

    图四十：核选「自动连结网址」的核选框。
  7. 「预设允许回响」选项会决定你的文章在预设的情况下，要不要让读者们能够发表回响。除非你要做成公告式的站台，否则开启回响功能可以说是利多于弊，因为这将让读者跟你有更好的互动，而这种互动则会吸引更多读者前来阅读你们所写下的东西。因此，请如图四十一，在这个选项选用预设的「开启」。

    图四十一：开启预设允许回响的选项。

这个页面主要是用来设定你所想采用的汇整方式，以及订定汇整档案的档名规则的。关于汇整的各项细节，你可以参考第八章的介绍。

  1. 在「汇整类别」这一栏中，你可以勾选你想使用的汇整方式；预设的汇整类别是单篇汇整以及逐月汇整，这意味著每一篇文章都会被单独做成一页，另外相同月份的所有文章也会被做成一页。这似乎是在兼顾节省硬碟使用以及提高便利性的双重考量下，相当不错的一种作法。
    由于每一份网志在刚设立的时候，是没有任何类别的，所以类别这一项并没有被核选。不过我们在后面的章节就会提到该如何运用分类汇整，所以现在不妨就按照图四十四这样，把类别给选起来。

修改个人数据

到此为止，你的身份仍旧叫做 Melody ，然后你的口令则是 Nelson 。很不幸地，全世界读过 Movable Type 文件的人都知道这回事。换而言之，这些陌生人都可以很快乐地联机到你的 Movable Type ，然后用这组预设的帐号跟口令登入，并大肆破坏。

没有人真的希望这种事情发生，所以请务必按照下列的步骤来把预设的帐号修改掉：

  1. 「使用者」字段中的是你用来登入 Movable Type 系统的使用者名字；预设的使用者是 Melody ，请随意修改成你自己想用的名字，如图四十七。这里不建议使用中文字，你最好还是祇使用英文字母跟数字来当作使用者名字；就算真的要用中文名字，也建议填到昵称那一栏去。

    图四十七：修改使用者字段，在这个范例中我们把 Melody 换成了 Jedi 。
  2. 「昵称」字段可填可不填；你的使用者名字不能跟其它使用者相同，但是昵称可以。所以遇到多位作者习惯用的名字重复时，就可以设在昵称里，而继续使用不同的使用者帐号来登入。另外，昵称当然也可以跟使用者名字相同，如图四十八。

    图四十八：填入昵称，在这个范例中填入的是 Jedi 。
  3. 「 Email 地址」这一栏虽然可以不填，但是你最好还是如图四十九加以填写；因为如果不幸你忘记自己的口令时， Movable Type 就得要知道你的 Email 地址，才有办法把新的口令寄给你。

    图四十九：填写 Email 地址，在这个范例中填入的是 JediMT@Jedism.org 。
  4. 如果你自己有另外的网站时，也可以如图五十在「网站网址」加以填写。

    图五十：填写网站网址，在这个范例中填入的是 http://blog.elixus.org/jedi/ 。
  5. 「偏好语言」不但会改变你的 Movable Type 的各项接口、讯息所使用的语言，甚至也会改变说明文件的语言，以及你所建立的新网志中的预设模版语言。请如图五十一从下拉式选单里选择 Taiwan Chinese （繁体中文）。

    图五十一：选择偏好语言，在这个范例中所选择的是 Taiwan Chinese 。
  6. 现在我们还要把预设的口令 (Nelson) 给换掉才安全。请在「口令」字段里填入你想要用的口令，大小写是不一样的。

    图五十二：填入新口令。
  7. 在「口令确认」字段里把你刚刚填入的口令再填一次，以确认你没有不慎打错字。

    图五十三：确认口令有没有打错。
  8. 如果你不慎忘记口令了， Movable Type 还可以用最后的口令恢复机制来让你挽回。在这种情况下， Movable Type 会询问你的出生地为何，这个时候你就得回答你在这边的「出生地」字段里所填写的东西。因此这一栏的内容你也应该保密，不要让其它人轻易得知（不过这一栏的答案就算外泄了，你还有机会保有秘密，因为 Movable Type 祇会把口令寄到你在步骤三所设定的 Email 地址去）。

    图五十四：填写恢复口令功能会用到的出生地信息，在这个范例中填写的是 Mars 。
  9. 以上的字段都填／改完后，请如图五十五按下「储存」按钮。

    图五十五：按下「储存」按钮。
  10. 个人数据储存起来后，你应该会看到如图五十六的讯息，提示你个人数据已经储存起来了。这个时候你的使用者名字、口令等信息都已经更新了，所以你毋须重新登入 Movable Type 。

    图五十六：你的个人数据皆已更新了。
